Getting started

Node.js is a platform built on Chrome’s JavaScript runtime for easily building fast, scalable network applications.
 
Gulp is a task manager, you can define different tasks to run certain commands. Those commands does the compilation job for JADE and LESS, and concatenates the JS files.
 
Bower is a dependency manager, it works by fetching and installing packages from all over, taking care of hunting, finding, downloading, and saving the stuff you’re looking for. Bower keeps track of these packages in a manifest file, bower.json.

MongoDB

Mongodb is the M in MEAN so it's required to run the application, please see here to download and learn how to install it on your system http://www.mongodb.org/

Installing tools

The following steps are intended to be an orientation guide, if you are not experienced with this you will need to learn more about it from Google :)
  • To install node and npm, go to http://nodejs.org/
  • Run npm install -g gulp bower to install Gulp and Bower
  • Download and install GIT for your platform http://git-scm.com/downloads

Starting the app

MeanJS uses ExpressJS as a server that can be started easily from command line.
 
Once you have the package ready to start working, 
 
- Move to the root folder of the project
- Run npm install (also will install bower deps)  
- Start MongoDB
- Run gulp so it builds the source and starts a local server at http://localhost:3000
 
If necessary, run bower install to ensure all dependecies are installed before run the application or you will receive console message related to missing assets.

Usage

Since this version is based on MeanJS and we've adapted Angle to work within this framework, the project is a seed application to add new modules by using generators described here http://meanjs.org/generator.html

Menu

The sidebar is different from the original version of Angle. It's doesn't load from a JSON file, it's used in the MeanJS way instead by using the service functions Menus.addMenuItem and Menus.addSubmenuItem

The function addMenuItem has a minor change to the param in order to make it compatible with the features Angle provides. 

Extra params supported has been added and they are optional:
iconClass: the class name for the menu item icon (default: fa fa-file-o)
translateKey: the translation key used to translate the menu item
alert: Any message to display next to the item (ideal for counters)
 
Example of file /modules/dashboard/client/config/dashboard.client.menu.js to demonstrate the use of the Menu service
 
(function() {
    'use strict';

    angular
        .module('app.dashboard')
        .run(coreMenu);

    coreMenu.$inject = ['Menus'];
    function coreMenu(Menus){

        Menus.addMenuItem('sidebar', {
            title: 'Dashboard',
            state: 'app.dash',
            type: 'dropdown',
            iconClass: 'icon-speedometer',
            position: 2,
            roles: ['*']
        });

        Menus.addSubMenuItem('sidebar', 'app.dash', {
            title: 'Dashboard v1',
            state: 'app.dashboard'
        });
        Menus.addSubMenuItem('sidebar', 'app.dash', {
            title: 'Dashboard v2',
            state: 'app.dashboard_v2'
        });
        Menus.addSubMenuItem('sidebar', 'app.dash', {
            title: 'Dashboard v3',
            state: 'app.dashboard_v3'
        });

    }

})();
 
Important: Note we use a main state for the main menu entry and that is used later for reference to associate each menu child with its corresponding parent.

Changes

We have implemented the following in order to adapt the Angle theme styles and script to the MeanJS framework.

- Implemented SCSS (sass) pipeline for files located under folder /modules/core/client/scss
- Used HTML Swig templates sinces they are of offcially supported by the MeanJS community
- The client core module is where main Angle assets takes place
- Changed the way sidebar works in order to use the core Menus service
- Changed server views to use Angle markup structure.
- Added minor changes to modules users and articles views and routes to match Angle markup requirements.
